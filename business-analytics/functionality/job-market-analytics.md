# Аналитика рынка труда

Фокусируемся на headhunter.ru

## Анализ количества вакансий и статистики зарплат

Суть:
- Собирать исторические данные по таким параметрам состояния рынка труда как количество вакансий и средняя зп.
- Собранная аналитика доступна публично, не требует авторизации.
- Админы могут управлять списком поисковых запросов для выборок, по которым строится аналитика.

Процесс:
- По планировщку делаем выборку вакансий по специальности для каждого поискового запроса из базы. [Язык запросов](https://hh.ru/article/1175) на headhunter позволяет гибко фильтровать вакансии, например для поиска позиций джава разработчика мы можем использовать следующий запрос - `NAME:(!"Java") NOT QA NOT AQA` (ищем "Java" в названии вакансии, убираем вакансии с "QA" или "AQA" в заголовке)
- Аггрегируем вакансии в data point - (айди поискового запроса, дата, количество вакансий, средняя зп)

Нефункциональные требования:
- Учесть лимиты по API у headhunter, ответственно отнестись к логированию/мониторингу ошибок от API

---

Идеи на будущее:
- Самые релевантные навыки по направлениям.
- Более узкие срезы для анализа, например поисковый запрос, плюс зп выше определенной суммы. Или количество вакансий не в РФ.
- Хранение самих вакансий. Для истории/анализа или рассылки уведомлений о появлении новых позиций.
