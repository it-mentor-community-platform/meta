# Интервальное повторение вопросов к собеседованиям

## Контекст

- У нас есть методички для повторения вопросов к собеседованиям. [Java](https://zhukovsd.github.io/java-backend-interview-prep/), [Python](https://zhukovsd.github.io/python-backend-interview-prep/). Это закрывает вопрос того, откуда брать вопросы
- У всех кто готовится к собесам, встаёт потребность повторять материал
- Существуют алгоритмы и формулы для расчёта интервалов при интервальном повторении, например [SM2](https://super-memory.com/english/ol/sm2.htm)

## Флоу работы с темами вопросов на интервальное повторение

- Пользователь заходит в раздел нашего приложения "интервальное повторение". Для него доступны все разделы с вопросами в виде списка. Категория первого уровня - название языка программирования, категория второго уровня - название темы. Например, "Java. Брокеры сообщений"
- Открыв тему, пользователь может ознакомиться со списком конкретных вопросов и ответов на них. Нажать "Добавить на повторение"
- Пользователь заходит в раздел "Вопросы на повторение". Там перечислены все категории, которые ранее были добавлены на повторение. Кнопка "повторить всё" переводит к флоу повторения вопросов из всех добавленных категорий. Альтернативно можно перейти к флоу повторения вопросов конкретной темы

## Флоу повторения вопросов

- Запуская данный флоу, пользователь имеет пул вопросов, включающий в себя одну или несколько тем
- Для каждого вопроса из пула, система знает дату последнего повторения и результат (знаю / не знаю)
- В соответствии с этой информацией формируется дата следующего повторения для каждого вопроса из пула. Если дата уже наступила - этот вопрос попадает в очередь на повторение
- Каждый вопрос из очереди по одному показывается пользователю. Так как наши вопросы по теории - не тест, а вопрос с открытым ответом, то пользователь изначально видит формулировку вопроса и кнопки с 3-5 вариантами того, насколько хорошо ответ запомнен, например - "отлично помню ответ", "помню ответ с пробелами", "отрывочно помню ответ", "не помню ответ"
- Пользователю в любой момент доступен просмотр "каноничного" ответа на вопрос из методички для самопроверки
- Исходя из того насколько хорошо пользователь помнит ответ, система обновит статус вопроса в пуле, и рассчитает новую дату повторения

## Аналитика

В дашборде интервального повторения пользователю доступна аналитика:

- Сколько вопросов "выучено". Выучил = ответил "помню" нужное количество раз
- Сколько вопросов в процессе изучения
- Аналогичная статистика доступна для каждой из добавленных категорий
- Статистика регулярности повторения вопросов
- Количество повторённых вопросов в каждый из дней за последние 365 дней
