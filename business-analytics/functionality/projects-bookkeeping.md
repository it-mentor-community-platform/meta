# Бизнес аналитика - учёт проектов

## Необходимый контекст

- Google Spreadsheet [таблица](https://docs.google.com/spreadsheets/d/1E66YrdvO7B_j0Ykge-JJDMtB1RfKhIzN_SsO7UPDbrU/edit?gid=0#gid=0), лист "projects". Пополняется ботом сообщества

## Способы добавления проектов

- Через интерфейс Telegram mini app
- Командой "/addproject" Telegram бота (текущий способ, используемый с 2024 года)
- Импорт из Google таблицы

## Хранилище проектов

- SQL база проекта
- Google таблица (текущий способ)

Подразумевается поддержка обоих хранилищ. SQL - для функционала проекта, таблица - для совместимости с уже существующим функционалом по учёту проектов и генерации [списка реализаций](https://zhukovsd.github.io/java-backend-learning-course/finished-projects/) на сайте роадмапа.

## Реакция системы на добавленный проект

- Оповещение членов чата @zhukovsd_it_chat
- Добавление проекта в БД, к которой будут обращаться клиентские API, работающие с проектами
- Добавление проекта в Google таблицу
- Любая бизнес логика, завязанная на новый проект - пересчёт бейджей ("ачивок") пользователя, например за количество сданных проектов; оповещение менторов, делающих ревью на данный проект
