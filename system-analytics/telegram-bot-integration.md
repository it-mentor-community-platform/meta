# Интеграция с Telegram ботом сообщества

Суть - Telegram бот сообщества, написанный на Python, решает ряд задач в чатах сообщества. Сдача проектов командой `/addproject`, использование AI-ассистента командой `/ai`.

Изначально предполагалось, что Telegram бот станет частью нашей общей backend-инфраструктуры (у хостинг провайдера Selectel, в московском дата центре) и будет общаться с сервисами проекта через Kafka топики. Связь двусторонняя. 

Однако, из-за ограничений РКН разворачивать Telegram бот'а на инфрастуктуре внутри РФ нет возможности - не хочется постоянно решать проблемы с обходом блокировок.

Решение следующее:
- Сервис-адаптер, развернутый внутри основной backend инфраструктуры в Selectel:
  - Слушает сообщения через кафка-консьюмеры, складывает их в outbox таблицу
  - Через публично доступное REST API, защищённое Basic Auth, даёт доступ к сохранённым сообщениям
  - Через публично доступное REST API, защищённое Basic Auth, даёт возможность создать сообщение, и сохраняет его в inbox таблицу
  - Отправляет сообщения из Inbox таблицы в соответствующие Kafka топики
- Telegram bot общается с REST API адаптера:
  - Шорт-поллит эндпоинты для чтения сообщений
  - По необходимости вызывает эндпоинты для отправки сообщений

На уровне gateway, правилами Spring Security настроено проксированние запросов к адаптеру без валидации JWT токена, потому что тут его не будет.
